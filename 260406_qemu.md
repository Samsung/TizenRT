### 2.3 `virtio_blk_test` Command Path

`virtio_blk_test` is exposed as a built-in testcase command for `qemu-virt`.

Implementation model:

- virtio-blk initializes during board bring-up
- the shell command calls `boardctl()`
- qemu-virt board code dispatches the ioctl to the virtio block self-test
- success prints `VIRTIO_BLK_TEST: PASS`
- failure prints `VIRTIO_BLK_TEST: FAIL ...`

This keeps the test command board-specific without directly hard-wiring TASH registration into kernel board init.

## 3. Docker Runtime Image

Runtime Docker support lives in:

- `tools/qemu_test/Dockerfile`
- `tools/qemu_test/run_qemu_tests.py`
- `build/configs/qemu-virt/tools/run_qemu.sh`

### Important runtime finding

Using stock Ubuntu `qemu-system-arm` directly caused boot failure because the guest observed:

- QEMU pflash erase size = `262144` bytes
- TizenRT partition layout uses small partitions like `bl1=32 KiB`, `bootparam=8 KiB`

That caused partition bring-up failure before TASH prompt.

To remove the external runtime dependency while keeping the same behavior, the runtime Dockerfile now builds a repo-owned image from:

- `ubuntu:24.04`
- official `qemu-10.2.0-rc3` source from `download.qemu.org`

During the image build, `hw/arm/virt.c` is patched in-place so `VIRT_FLASH_SECTOR_SIZE` changes from `256 * KiB` to `4 * KiB`, and the build log verifies the replacement before compilation.

This reproduces the flash geometry required by the qemu-virt partition map and SMARTFS/userfs flow without depending on `pcs1265/qemu-virt-flash-4k:1.0.0`.

Runtime verification also checked `info qtree` in the rebuilt image and confirmed:

- `cfi.pflash01 num-blocks = 16384`
- `cfi.pflash01 sector-length = 4096`

The local runtime image built in this workspace is:

- `tizenrt-qemu-virt-runtime:local`

## 4. Local Validation Cycle

### 4.1 Commands executed

Build/configure/download:

```bash
./os/dbuild.sh configure qemu-virt loadable_xip_elf
./os/dbuild.sh build
./os/dbuild.sh download bl1 kernel common app1 bootparam
```

Runtime image:

```bash
docker build -t tizenrt-qemu-virt-runtime:local -f tools/qemu_test/Dockerfile .
```

Fresh flash refresh before rerun:

```bash
docker run --rm \
  --mount "type=bind,source=$PWD,target=/workspace" \
  -w /workspace \
  tizenrt/tizenrt:2.0.1 \
  bash -lc "TOPDIR=/workspace bash build/configs/qemu-virt/qemu-virt_download.sh ERASE && \
            TOPDIR=/workspace bash build/configs/qemu-virt/qemu-virt_download.sh bl1 kernel common app1 bootparam"
```

Runtime harness:

```bash
docker run --rm --network none \
  -v "$PWD:/workspace" \
  -w /workspace \
  tizenrt-qemu-virt-runtime:local \
  python3 tools/qemu_test/run_qemu_tests.py \
    --artifact-dir /workspace/runtime-results \
    --boot-timeout 120 \
    --command-timeout 15 \
    --tc-timeout 180
```
